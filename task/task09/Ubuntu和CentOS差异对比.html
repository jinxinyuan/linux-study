ubuntu和centos的区别

Ubuntu和CentOS都是非常流行的发行版，工作中可能都会涉及到，甚至要同时使用Ubuntu和CentOS。这里把Ubuntu和CentOS的区别做一些对比，方便查阅。

发行公司和发行版本

Ubuntu
Ubuntu由Canonical公司发行的开源操作系统。中文官网为cn.ubuntu.com。Ubuntu每年发布两个版本，分别在4月份和10月份，以年份和月份命名，比如Ubuntu 18.04，Ubuntu 19.10。

每两年的4月份发行长期支持版（LTS），提供5年的技术支持，不过从2018年IBM收购RedHat后，Ubuntu18.04更改了支持策略，提供长达10年的技术支持。

CentOS
RedHat公司的CentOS是RedHat版本的一个开源分支，由社区驱动。每个主要发行版本提供10年技术支持，现在RedHat已被IBM收购。

软件包管理

Ubuntu基于Debian开发，和Debian使用相同的包管理器，CentOS/RedHat是同样的包管理器。

Ubuntu CentOS
本地包管理命令 dpkg rpm
网络包管理命令 apt，从软件源安装，可以卸载本地包 yum，从软件源安装，可以卸载本地包

使用apt和yum都支持update和upgrade进行更新升级，但是apt update仅仅是获取需要更新的信息，之后要使用apt upgrade进行系统和软件包的更新升级，而yum update和upgrade基本上是一致的，直接就进行系统和软件包的升级。

用户和组

对用户和组的管理，有这样几个命令：adduser，addgroup，useradd，groupadd。感觉有点乱吧，到底该用哪个？

在CentOS上，目前只有useradd和groupadd，而在Ubuntu上全部存在。不过在Ubuntu上推荐使用adduser和addgroup。

在Ubuntu/Debian系列的发行版上，adduser和addgroup是比较新的命令，联机文档也给出了建议，使用adduser和addgroup代替useradd和groupadd。

所以，简单总结如下表：

Ubuntu CentOS
创建用户 adduser useradd
创建用户组 addgroup groupadd
删除用户 deluser userdel
删除用户组 delgroup groupdel

配置文件

/etc包含了系统和软件的配置文件、系统服务控制脚本等。目录组织结构上，有一些区别。bash的默认配置文件名称有差别。

bash配置文件

在Ubuntu上开始新的会话，bash运行除了/etc/下的配置文件，还会去读取当前用户主目录下的.profile以及.bashrc文件，之后运行新的bash，会读取.bashrc。.profile在CentOS上，换成了.bash_profile。

网络配置

Ubuntu18.04使用了netplan作为网络配置管理。配置文件在/etc/netplan。而在Ubuntu16.04以及之前的版本，配置文件在/etc/network。CentOS的网络配置在/etc/sysconfig/network-scripts/目录。

VirutalBox虚拟机配置静态IP

虚拟机使用DHCP进行动态的分配IP地址，在只有一个虚拟机系统的情况下，一般IP地址也不会变化，但是如果需要在同一台设备上同时启动多个虚拟机，IP地址往往会发生变化，这时候设置静态IP就比较方便。

Ubuntu18.04设置静态IP

配置文件名称会有所不同，但是都在/etc/netplan目录下，是.yaml文件。

network:
    ethernets:
        enp0s3:
            addresses: []
            dhcp4: true
            optional: true
        enp0s8:
            addresses:
              - 192.168.56.102/24
            dhcp4: true
            optional: true

配置完成后，启用网络设置：
sudo netplan apply

CentOS配置静态IP

在本机运行的VirtualBox虚拟机上，配置文件是/etc/sysconfig/network-scripts/ifcfg-enp0s8

TYPE=Ethernet
PROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=dhcp
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_FAILURE_FATAL=no
IPV6_ADDR_GEN_MODE=stable-privacy
NAME=enp0s8
UUID=5b831cbd-831e-42f3-a17a-d3125fc960c8
DEVICE=enp0s8
ONBOOT=yes
#最后两行是单独加入的用于配置静态IP
IPADDR=192.168.56.112
NETMASK=255.255.255.0

配置完成后，可以通过SSH客户端使用192.168.56.112连接。配置完成后，重启网络服务：
sudo systemctl restart network.service
